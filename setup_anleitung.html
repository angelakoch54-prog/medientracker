Firebase Setup

Diese Anleitung zeigt Angela Schritt fuer Schritt, wie sie Firebase einrichtet, um ihre Mediensammlung zwischen iPhone und iPad zu synchronisieren.

Was du bekommst: Kostenloses Firebase-Konto → Echtzeit-Sync zwischen all deinen Geraeten → Automatisches Backup in der Cloud

Schritt 1: Firebase-Konto erstellen

1

Gehe zu console.firebase.google.com

Melde dich mit deinem Google-Konto an (oder erstelle kostenlos eines).

Schritt 2: Neues Projekt erstellen

2
  1. Klicke auf “Projekt erstellen” (grosses Plus-Symbol)
  2. Gib einen Projektnamen ein, z.B. MedienTracker
  3. Google Analytics: kann deaktiviert werden (nicht benoetigt)
  4. Klicke auf “Projekt erstellen”
  5. Warte bis das Projekt erstellt ist, dann auf “Weiter”

Schritt 3: Web-App hinzufuegen

3
  1. Im Projekt-Dashboard klicke auf das “</>” (Web)-Symbol
  2. App-Spitzname eingeben: z.B. MedienTracker Web
  3. Kein Hosting einrichten (Haekchen NICHT setzen)
  4. Klicke auf “App registrieren”
  5. Du siehst jetzt deinen firebaseConfig-Code
Wichtig: Kopiere den gesamten Config-Block - du brauchst ihn in Schritt 5!

Der Code sieht ungefaehr so aus:

const firebaseConfig = { apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXX", authDomain: "meinprojekt.firebaseapp.com", projectId: "meinprojekt", storageBucket: "meinprojekt.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef123456" };

Schritt 4: Firestore einrichten

4
  1. Im linken Menue unter “Erstellen” auf “Firestore Database” klicken
  2. Klicke auf “Datenbank erstellen”
  3. Waehlbar: “Im Testmodus starten” (einfacher fuer den Anfang)
  4. Region: europe-west3 (Frankfurt) waehlen
  5. Klicke auf “Weiter” und dann “Aktivieren”

Sicherheitsregeln setzen

Damit nur du auf deine Daten zugreifen kannst:

  1. In der Firestore-Datenbank auf “Regeln” klicken
  2. Den bestehenden Text ersetzen mit:
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /users/{userId}/{document=**} { allow read, write: if request.auth != null && request.auth.uid == userId; } } }
  1. Klicke auf “Veroeffentlichen”

Schritt 5: Anonymous Auth aktivieren

5
  1. Im linken Menue unter “Erstellen” auf “Authentication” klicken
  2. Klicke auf “Erste Schritte”
  3. Unter “Anmeldemethoden” klicke auf “Anonym”
  4. Schalte die Option “Aktivieren” ein
  5. Klicke auf “Speichern”

Schritt 6: Konfiguration in die App eintragen

6

Oeffne die Datei js/firebase-sync.js in einem Texteditor (z.B. Editor oder Notepad++).

Suche diesen Abschnitt (ganz oben):

const FIREBASE_CONFIG = { apiKey: "DEINE_API_KEY", authDomain: "DEIN_PROJEKT.firebaseapp.com", projectId: "DEIN_PROJEKT_ID", storageBucket: "DEIN_PROJEKT.appspot.com", messagingSenderId: "DEINE_SENDER_ID", appId: "DEINE_APP_ID" };

Ersetze die Platzhalter mit den Werten aus Schritt 3:

PlatzhalterErsetzen durch
DEINE_API_KEYapiKey aus Firebase Console
DEIN_PROJEKT.firebaseapp.comauthDomain aus Firebase Console
DEIN_PROJEKT_IDprojectId aus Firebase Console
DEIN_PROJEKT.appspot.comstorageBucket aus Firebase Console
DEINE_SENDER_IDmessagingSenderId aus Firebase Console
DEINE_APP_IDappId aus Firebase Console

Speichere die Datei und lade die App neu.

Schritt 7: App auf Webserver / iPhone installieren

7
Die App muss auf einem Webserver laufen (nicht einfach als lokale Datei oeffnen), damit der Service Worker und Firebase korrekt funktionieren.

Option A: Kostenloser Firebase Hosting (empfohlen)

  1. Im Firebase-Dashboard links auf “Hosting” klicken
  2. Klicke auf “Erste Schritte”
  3. Firebase CLI installieren (erfordert Node.js):
    npm install -g firebase-tools
  4. Im MedienTracker-Ordner:
    firebase login firebase init hosting firebase deploy
  5. Du bekommst eine URL wie: meinprojekt.web.app

Option B: GitHub Pages (kostenlos, kein Node noetig)

  1. Erstelle ein kostenloses Konto auf github.com
  2. Erstelle ein neues Repository (z.B. medientracker)
  3. Lade alle MedienTracker-Dateien hoch
  4. In Repository-Einstellungen: Pages → Deploy from branch → main
  5. Deine App ist erreichbar unter: deinname.github.io/medientracker

Option C: Lokaler Entwicklungsserver (fuer Testen)

Falls Python installiert ist:

cd C:\Users\info\Desktop\MedienTracker python -m http.server 8080

Dann im Browser oeffnen: http://localhost:8080

Schritt 8: App auf iPhone/iPad installieren

8
  1. Oeffne die App-URL in Safari auf dem iPhone/iPad
  2. Tippe auf das Teilen-Symbol (Viereck mit Pfeil nach oben)
  3. Scrolle runter und tippe auf “Zum Home-Bildschirm hinzufuegen”
  4. Gib den Namen MedienTracker ein
  5. Tippe auf “Hinzufuegen”
Die App erscheint jetzt als echte App auf dem Homescreen und laeuft im Vollbildmodus ohne Safari-Leisten!

Optional: OMDb API Key (fuer DVD-Daten beim Barcode-Scan)

  1. Gehe zu omdbapi.com/apikey.aspx
  2. Waehle “FREE! (1,000 daily limit)”
  3. Gib deine E-Mail ein und bestatige
  4. Oeffne js/api.js und ersetze:
    let OMDB_KEY = 'DEIN_OMDB_KEY';
    mit deinem echten Key.

Ohne OMDb-Key funktioniert die Buchsuche (Google Books) weiterhin kostenlos und ohne Registrierung.

Haeufige Fragen

Werden meine Daten sicher gespeichert?

Ja. Firebase speichert Daten verschluesselt in Google-Rechenzentren. Mit den Sicherheitsregeln aus Schritt 4b kann nur du selbst auf deine Daten zugreifen.

Was passiert wenn ich offline bin?

Die App speichert alles lokal auf dem Geraet (IndexedDB). Sobald du wieder online bist, werden die Aenderungen automatisch synchronisiert.

Kann ich die App auf mehreren Geraeten nutzen?

Ja! Installiere die App auf iPhone und iPad, trage dieselbe Firebase-Config ein, und alle Aenderungen werden in Echtzeit synchronisiert.

Ist Firebase kostenlos?

Der Spark-Plan (kostenlos) reicht fuer eine persoenliche Mediensammlung voellig aus. Er erlaubt 1 GB Daten und 50.000 Lesezugriffe pro Tag.

Wie sichere ich meine Daten?

Firebase macht automatisch Backups. Zusaetzlich kannst du in der Firebase Console unter Firestore alle Daten exportieren.
